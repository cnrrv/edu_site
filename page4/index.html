Неявная адресация в языке С

Язык программирования С, как и ассемблер, относится к низкоуровневым языка программирования и предоставляет ряд методов для управления памятью. Благодаря этому у разработчика есть возможность контролировать количество потребляемой памяти на различные процессы, не выходя за пределы резервов памяти компьютера.

Неявная адресация

    В традиционных языках программирования cуществуют три вида памяти: статический, стековый и динамический. При работе с ними используется неявная адресация. Для статической и динамической памяти используется непосредственный метод адресации, а для стековой памяти - стековый метод.

Непосредственная адресация

Статическая память выделяется на стадии компиляции и сборки. Переменные, определенные в программе статическим методом, имеют фиксированный адрес, известный до запуска программы и не изменяются в процессе её работы. К статическим переменным относятся глобальные переменные и статические переменные

функция summ, которая подсчитывает сумму статической переменной x и глобальной переменной y

int y = 20;

int summ() {

  return x + y;

Концепция динамического выделения памяти в языке С позволяет выделять память во время выполнения программы. Такой метод применяется в случаях, когда заранее неизвестно количество данных, которое будет использоваться в программе. При работе с динамической памятью используют указатели - переменные, которые хранят в себе адрес памяти объекта.

Стековая адресация

Переменная, создаваемая внутри функции, называется локальной и определяется с помощью стековой памяти. Стек - это структура данных, представляющая собой список элементов, организованный по принципу «последним пришел - первым вышел». Структуру стека можно сравнить со стопкой книг. Когда внутри функции инициализируется переменная, она кладётся на вершину стека. Затем, когда функция завершает работу, все переменные, связанные с этой функцией, удаляются, а стек очищается.

  int y = 20;

  return x + y;

Косвенная адресация позволяет задействовать в программе больший объем памяти по сравнению с прямой адресаций, но требует дополнительных вычислений для нахождения эффективного адреса и более частое обращение к памяти.
