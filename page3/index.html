Явная адресация в языке ассемблера

Концепция современных языков программирования основана на том, чтобы упростить работу с памятью и избавить разработчиков от её ручного управления. Тем не менее, до сих пор при разработке IT-продуктов используются низкоуровневые языки программирования, позволяющие работать с памятью напрямую. К такому виду языков относится язык ассемблера.

Явная адресация

Для прямой адресации памяти используется два операнда: один содержит в себе ячейку памяти, а второй ссылается на регистр. Квадратные скобки означают обращение по адресу, указанному внутри этих скобок. Для доступа к данным требуется только одна операция обращения к памяти. Прямая адресация считается самым простым видом адресации операнда, так как эффективный адрес содержится в его команде и для выполнения операции не требуется дополнительных преобразований.

mov DX, [1303]

поместить в регистр DX значение из памяти по адресу 1303

mov EAX, sum

поместить в регистр EAX значение, хранящееся по адресу sum - в процессе компиляции ассемблер заменит sum на соответствующий адрес, хранящийся в сегменте данных

Во время косвенной адресации применяется возможность компьютера использовать адресацию вида сегмент: смещение. Косвенная адресация используется для структур данных, содержащих несколько элементов (стеки, массивы). Для доступа к данным требуются две ссылки: первая - для получения эффективного адреса, вторая для доступа к данным

add AX, [BX+2]

прибавить к значению, находящемуся в регистре AX значение, находящееся по адресу регистра BX со смещением 2

mov EAX, [ESI][EDX]

переместить в регистр EAX значение, находящееся по адресу ESI+EDX

Косвенная адресация позволяет задействовать в программе больший объем памяти по сравнению с прямой адресаций, но требует дополнительных вычислений для нахождения эффективного адреса и более частое обращение к памяти.
