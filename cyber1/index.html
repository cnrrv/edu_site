Безопасность операционной системы: разработки

Раздел 1. Безопасность информации в ОС

В операционной системе специального назначения Astra Linux используются встроенные средства защиты ОС. Средства защиты информации нужны для того, чтобы обеспечивать реализацию принципов информационной безопасности: конфиденциальности, доступности и целостности.

В Astra Linux используются следующие встроенные способы защиты информации:

Дискреционные права доступа

Это управление доступом субъектов к объектам с помощью списков управления доступом или посредством изменения прав доступа объекта напрямую его владельцем. Под субъектом понимается пользователь, учетная запись. Под объектом доступа понимается файл, каталог, или другой документ файловой системы.

Мандатный контроль целостности

В нем применяются метки конфиденциальности для объектов доступа и формы допуска для субъектов доступа. Основные правила мандатного контроля целостности: субъектам доступно чтение только тех объектов, чей уровень безопасности не выше уровня субъекта. Аналогичное правило работает для записи объекта.

Системный киоск

Это программа, которая используется для ограничения среды пользователя. Программа позволяет создавать и настраивать системные профили и профили для пользователей. Для запуска требуются права администратора.

Раздел 2. Учетные записи пользователей и групп

Для безопасной работы в системе используют различные модели безопасности, позволяющие работать с конфиденциальными данными только лицам с определенным уровнем полномочий и закрывать доступ для остальных пользователей. Одной из таких моделей является дискреционное управление доступом.

Дискреционное управление доступом (Discretionary Access Control, DAC) – это разграничение доступа между субъектами и объектами системы. Субъектом обычно выступает учетная запись, пользователь или сущность, выполняющая действия с объектами доступа. В то же время объект доступа – это единица информационной системы (файл, документ, каталог и т.п.), доступ к которой может быть ограничен. В дискреционной модели управления доступом владелец объекта сам устанавливает и меняет права субъектов. Изучим, как дискреционное управление доступом применяется в ОС Astra Linux на базовом уровне сложности («Орёл»).

Каждой учетной записи сопоставляются уникальное в рамках системы символьное имя пользователя или имя группы и уникальный числовой идентификатор пользователя (user ID, UID) или группы (group ID, GID). На рисунке ниже родемонстрированы действующие UID и GID пользователя roman, полученные с помощью утилиты id. Под номером 1 представлена информация о UID пользователя – 1200 и имя пользователя - roman. Под номером 2 указана информация о GID первичной группы пользователя – 1200, и имя первичной группы – roman. Под номером 3 выводится информация о группах, в которых состоит пользователь. В данном случае первая группа при перечислении является первичной группой, а все группы после нее – вторичными.

Полезные утилиты:

Для получения прав суперпользователя используется команда:

sudo su

Для создания пользователя используется следующая утилита:

useradd [имя_пользователя]

Вывести на экран терминала UID и GID пользователя:

id [имя_пользователя]

Для изменения UID пользователя используется команда:

usermod -u [число] [имя_пользователя]

Поменять GID группы:

groupmod -g [число] [имя_группы]

Команда для создания группы:

addgroup [имя_групппы]

Добавить пользователя во вторичную группу:

usermod -G [имя_группы] [имя_пользователя]

Изменить командный интерпретатор пользователя:

usermod -s [путь_до_интерпретатора] [имя_пользователя]

Добавить комментарий пользователя:

usermod -c «комментарий» [имя_пользователя]

Раздел 3. Файлы, каталоги и дискреционные права доступа к ним

Чтобы не происходило ситуаций, когда у пользователя есть доступ к файлам, которые не предназначены для изменения, чтения или удаления, в Astra Linux используется модель дискреционного управления доступом, которая включает в себя права доступа и списки управления доступом.
Для каждой из этих категорий учетных записей пользователей задаются три вида прав доступа к сущностям:
– право на чтение (read, r);
– право на запись (write, w);
– право на выполнение (execute, x).
На рисунке ниже можнo увидеть атрибуты файла paul1.txt в короткой записи. Разберем ее подробнее.

Первый символ определяет тип сущности (в данном случае первое тире означает, что paul1.txt - это файл). Следующие 9 символов означают права доступа. Их следует разделять по тройкам. Первая тройка – права доступа к файлу для учетной записи пользователя-владельца paul («rw-» означает, что у пользователя-владельца есть права доступа на чтение и запись файла), вторая тройка – права доступа к файлу для учетных записей, входящих в группу-владельца сущности atreides («rw-» - учетные записи группы-владельца имеют право доступа на чтение и запись файла), третья тройка означает права доступа к файлу для остальных учетных записей системы («r--» означает, что у остальных пользователей есть право чтения файла). Следующий элемент, отделяемый пробелом – количество ссылок на файл в ОС (1). После числового значения идёт имя учетной записи владельца-сущности (paul). Следующее имя – группа-владелец сущности (atreides). Следующее число – размер файла в байтах (22). Далее идут данные о дате создания (фев 5 22:44) и само имя файла (paul1.txt).

Полезные утилиты:

Для навигации в терминале используется команда:

cd [путь_до_каталога]

Для создания файла используется следующая утилита:

touch [имя_сущности]

Вывести на экран терминала содержимое файла:

сat [имя_сущности]

Для создания каталога используется утилита:

mkdir [название_каталога]

Для удаления файла используется команда:

rm [имя_сущности]

Для просмотра атрибутов каталога можно использовать команду:

ls -ld [путь_до_каталога]

Для задания прав доступа на сущность используется команда:

chmod [категория][оператор][права_доступа] [имя_сущности]

Для изменения владельца и группы владельца применяется команда:

sudo chown [владелец]:[группа-владелец] [имя_сущности]
